# 校友有点闲 演示脚本

## 演示目标

展示一个完整的校园二手交易闭环：

1. 用户发布商品
2. 管理员审核商品
3. 买家下单购买
4. 卖家发货
5. 买家确认收货
6. 用户互动与平台治理能力，包括收藏、留言、消息通知、举报和信用展示

## 演示前准备

1. 启动后端：backend 目录执行 mvn spring-boot:run，确认后端端口为 8081
2. 启动前端：frontend 目录执行 npm.cmd run dev，前端当前默认访问端口为 5173
3. 确认数据库已有默认管理员：admin / your_demo_password
4. 确认数据库已有两个普通演示账号：buyer1001 / your_demo_password、buyer2001 / your_demo_password

## 演示顺序

## 一、普通用户发布商品

1. 登录普通用户 buyer1001 / your_demo_password
2. 进入发布商品页
3. 上传商品图片
4. 填写商品标题、分类、价格、库存、描述
5. 提交发布
6. 强调：商品状态此时进入待审核，不会立即上架

## 二、管理员审核商品

1. 退出普通用户，登录管理员 admin / your_demo_password
2. 进入顶部导航中的“商品审核”页面
3. 在商品审核表格中找到新发布的商品
4. 点击通过
5. 强调：审核通过后，商品才会出现在首页商品列表中

## 三、第二个买家购买商品

1. 退出管理员，登录 buyer2001 / your_demo_password
2. 在首页搜索新商品标题
3. 进入商品详情页
4. 加入购物车并提交订单
5. 强调：系统会校验库存和卖家归属，一次订单只允许同一卖家的商品

## 四、卖家发货

1. 退出 buyer2001，重新登录卖家 buyer1001 / your_demo_password
2. 进入订单页，切到卖家订单
3. 找到刚生成的订单
4. 点击发货

## 五、买家确认收货

1. 再切回 buyer2001 / your_demo_password
2. 进入订单页，查看买家订单
3. 点击确认收货
4. 强调：订单状态完成闭环，交易流程结束

## 演示时可顺带强调的技术点

1. 前后端分离：Vue3 + Vite 对接 Spring Boot
2. JWT 登录鉴权：登录后带 Authorization 头访问业务接口
3. MyBatis-Plus：快速完成分页、查询和 CRUD
4. 多图商品：商品支持多图上传与展示
5. 管理端审核流：商品需要管理员审核后上架
6. 订单状态流转：待发货 -> 已发货 -> 已完成，对应数据库状态 CREATED -> SHIPPED -> FINISHED
7. 用户与角色控制：普通用户与管理员页面和权限不同
8. 特色分类专区：管理员可维护特色分类，并把商品分配到首页专题展示区
9. 用户资料体系：支持公开资料页、交易偏好和隐私开关
10. 收藏体系：支持商品收藏与个人收藏管理
11. 留言与回复：支持商品详情页留言、递归回复和消息通知
12. 平台治理：支持商品举报、后台举报处理与用户信用统计

## 六、特色分类演示

1. 管理员登录后台
2. 进入“特色分类管理”页面，确认“毕业甩卖”“新生必备”等特色分类处于启用状态
3. 进入“商品审核”页面，在待审核商品上勾选特色分类并点击“通过”
4. 如需调整已上架商品，可进入“商品管理”页面查看商品信息，再回到“商品审核”页按“在售”状态筛选并点击“保存特色”
5. 返回首页查看“特色专区”是否展示该商品

## 七、后台管理结构演示

1. 管理员登录后进入“管理后台”总览页
2. 展示用户总数、商品总数、订单总数、待审核商品数量
3. 展示近 7 天趋势图和分类分布图
4. 依次说明六个入口卡片：商品审核、商品管理、用户管理、留言管理、举报管理、特色分类管理
5. 强调：具体管理操作已经拆分到独立页面，后台首页负责总览和导航

## 八、收藏与资料展示演示

1. 普通用户登录后进入任意商品详情页
2. 点击“收藏商品”
3. 从顶部导航进入“我的收藏”查看收藏列表
4. 点击卖家昵称或卖家资料入口，进入公开资料页
5. 展示公开资料中的信用分、信用等级、成交数量和交易偏好

## 九、留言与消息通知演示

1. 买家在商品详情页留言咨询
2. 切换卖家账号，在同一商品详情页点击“回复”进行回复
3. 再切回买家账号，查看“消息中心”是否收到新回复提醒
4. 展示消息中心的未读状态、时间双行显示和删除能力
5. 强调：留言与回复形成了接近评论区的递归互动模式

## 十、举报与后台处理演示

1. 买家在商品详情页点击“举报商品”并提交举报
2. 管理员进入“举报管理”页面查看举报记录
3. 选择处理结果，例如下架商品或忽略举报
4. 买家或卖家可在消息中心收到举报处理结果通知，并进入举报详情页查看处理结果
5. 强调：举报详情页不展示举报人信息，保护举报匿名性
6. 返回公开资料页或后台用户管理页，展示信用统计和治理结果的联动
